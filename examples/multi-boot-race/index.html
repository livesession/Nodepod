nodepod - multi-boot race (reproduces #39)

Reproduces: "when booting multiple Nodepod instances at once, the boot promises sometimes never resolve". Click Run to fire N Nodepod.boot() calls in parallel via Promise.allSettled(). Each boot is wrapped in a BOOT_TIMEOUT_MS watchdog, any boot that doesn't settle within that window is flagged as HUNG. Adjust the instance count and re-run, higher counts (4+) make the hang much more likely to appear.
